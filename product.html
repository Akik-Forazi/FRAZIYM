Architecture

Built to run everything

A layered system: unified backend abstraction → typed tool layer → loop engine → streaming UI. Each layer is independently testable and swappable.

LAYER 4
CLI + Web UI
Interactive REPL, SSE streaming dashboard, model manager, file explorer
LAYER 3
Loop Engine
Plan → Execute → Verify → Replan. Goal mode with multi-cycle orchestration
LAYER 2
Agent Core
Native tool calling + fallback text parsing. Loop detection, memory, history
LAYER 1
Backend Engine
8+ providers via unified interface. Factory pattern, auto-detection, health checks
Inference Backends

Your model, your rules

🦙

Ollama

Local server with native tool calling. Default backend. Supports all Ollama-registered models including Gemma 3, Llama 3.2, Qwen 2.5, Phi-3.5, and more.

🤖

OpenAI-Compatible

Covers OpenAI, LM Studio, Groq, Together AI, Mistral, DeepInfra, vLLM — any server exposing the /v1/chat/completions API.

🧠

Anthropic & Gemini

Claude (Sonnet, Opus, Haiku) and Google Gemini Flash/Pro via their official APIs. Full SSE streaming with proper message formatting.

📦

llama-cpp-python

Run GGUF files directly in-process. GPU offloading with n_gpu_layers, configurable context window, real-time token streaming.

🤗

HuggingFace

Load any .safetensors, .bin, or .pt model from a local directory or HF repo. Supports 4-bit and 8-bit quantization via bitsandbytes.

⚙️

KoboldCpp / TextGen

Compatibility layer for community servers. Falls back to OpenAI-compatible endpoint if available.

Research Framework
v0.2.4 Technical Report

XORZEN — Sparse Adaptive Intelligence

A paradigm shift in LLM architecture achieving state-of-the-art cognitive performance at a fraction of traditional costs. Developed by FRAZIYM AI.

1. HASS Architecture

The Hybrid Attention-Shard Switch (HASS) block replaces standard self-attention with a multi-modal manifold containing three distinct pathways:

  • Locality-Optimized: High-precision local context processing.
  • Low-Rank Global: Compressed long-range dependencies.
  • Temporal State Space (TSSK): Linear-time sequential processing.

2. Adaptive Sparse Routing

XORZEN makes real-time routing decisions for every token, resulting in a model that is 90% sparse during inference.

  • Dynamic Compute: Evaluates "Cognitive Load" per token.
  • Active Depth: Variable layer stack resolution (16–24 layers).
  • Memory Wall Solution: Disk-sharded Mixture-of-Experts (MoE).

3. Latent Chain-of-Thought (L-CoT)

Internal Monologue Reasoning across six cognitive components: Intention · Decomposition · Confidence · Contradiction · Direction · Summary.

Execution Modes

Three ways to run

axonix --lc

Interactive Mode

Full REPL with agent/chat/goal sub-modes. Switch models, inspect memory, and run shell commands with !-prefix — without leaving the session.

axonix --goal "…"

Goal Mode

The Loop Engine decomposes your goal into atomic sub-tasks, executes each, verifies success with an LLM validator, and replans on failure — up to 5 full cycles.

axonix --web

Web UI Mode

Launches the full browser dashboard at localhost:7860. Can be combined with --lc or --goal to run both simultaneously.
